## 시나리오별 상세 결과

### SCN-01: 신규 회원가입

- **결과**: ✅ PASS
- **검증 항목**
  - `/signup` 페이지 접근 및 폼 입력 가능
  - 이름·이메일·비밀번호 조건 입력 후 제출
  - 서버 201 응답 후 `/login` 으로 리다이렉트됨
- **비고**: 성공 토스트 메시지("회원가입이 완료되었습니다")가 짧게 표시 후 사라져 스냅샷에 미포착됨. 리다이렉트 동작으로 성공 확인.

---

### SCN-02: 이메일 중복으로 인한 회원가입 실패

- **결과**: ✅ PASS
- **검증 항목**
  - 동일 이메일(`scntest1@example.com`)로 `/signup` 재시도
  - 서버 409 Conflict 응답 (콘솔 에러 확인)
  - `/signup` 페이지 유지, 폼 데이터 보존
  - 중복 레코드 미생성 확인
- **비고**: 오류 토스트가 빠르게 사라져 정적 스냅샷 미포착. 페이지 유지 + 콘솔 에러로 동작 확인.

---

### SCN-03: 로그인 및 할일 목록 진입

- **결과**: ✅ PASS
- **검증 항목**
  - 올바른 자격증명으로 로그인 시 `/` 이동
  - 인증된 상태에서 `/login` 접근 시 자동으로 `/` 리다이렉트 (PublicRoute 동작)
  - 할일 목록 페이지 정상 렌더링

---

### SCN-04: 로그인 실패 및 재시도

- **결과**: ✅ PASS
- **검증 항목**
  - 잘못된 비밀번호(`WrongPass999`) 입력
  - 서버 401 응답 (콘솔 에러 확인)
  - `/login` 페이지 유지, 폼 데이터 보존
- **비고**: 오류 토스트가 빠르게 사라져 스냅샷 미포착. 페이지 유지로 동작 확인.

---

### SCN-05: 개인정보(이름) 수정

- **결과**: ✅ PASS
- **검증 항목**
  - `/profile` 페이지에서 이름 필드에 현재 이름 자동 로드 ("김민영")
  - "박민영"으로 수정 후 "이름 저장" 클릭
  - 서버 200 응답
  - 헤더 영역의 사용자 이름이 "박민영"으로 즉시 갱신됨 ✓

---

### SCN-06: 비밀번호 변경

- **결과**: ✅ PASS
- **검증 항목**
  - 현재 비밀번호 `Planning123`, 새 비밀번호 `NewPlan456` 입력
  - "비밀번호 변경" 클릭 후 폼 초기화됨 (성공 처리 확인)
  - 이후 새 비밀번호(`NewPlan456`)로 인증 흐름 유지 확인
- **비고**: 성공 토스트 빠르게 사라짐. 폼 초기화 + 이후 인증 유지로 확인.

---

### SCN-07: 할일 등록 (날짜 포함)

- **결과**: ✅ PASS
- **검증 항목**
  - `/todos/new` 접근
  - 제목("디자인 시안 v1 제출"), 카테고리("업무"), 설명, 시작일·종료일 입력
  - "저장하기" 후 `/` 리다이렉트
  - 목록에 새 할일 표시 확인 ✓

---

### SCN-08: 잘못된 날짜 범위로 할일 등록 실패

- **결과**: ✅ PASS
- **검증 항목**
  - 시작일 `2026-05-20`, 종료예정일 `2026-05-15` 입력 (종료일 < 시작일)
  - 제출 시 `/todos/new` 유지
  - 오류 메시지 "종료 예정일은 시작일(2026-05-20) 이후여야 합니다." 표시 ✓
  - 폼 데이터 유지 확인 ✓

---

### SCN-09: 할일 수정

- **결과**: ✅ PASS
- **검증 항목**
  - 목록의 수정 버튼 클릭 → `/todos/:id/edit` 이동
  - 기존 데이터(제목, 카테고리, 설명) 자동 로드 확인 ✓
  - 종료예정일 변경(`2026-05-17`), 카테고리 변경("업무"→"개인")
  - 저장 후 `/` 리다이렉트, 변경사항 목록 반영 ✓
- **비고**: 1차 시도 시 시작일 미입력으로 저장 실패. 시작일 포함 2차 시도에서 성공. 필수 필드 검증 동작 확인됨.

---

### SCN-10: 할일 완료 처리 및 토글

- **결과**: ✅ PASS
- **검증 항목**
  - 체크박스 클릭 → `[checked]` 상태로 변경 (완료 처리) ✓
  - 재클릭 → 체크 해제 (미완료 복원) ✓
  - BR-03 소유권 검증: 본인 소유 할일만 표시됨으로 간접 확인

---

### SCN-11: 할일 삭제

- **결과**: ✅ PASS
- **검증 항목**
  - 삭제 버튼 클릭 → 확인 다이얼로그 표시 ✓
  - 다이얼로그 메시지: "**디자인 시안 v1 제출** 을(를) 삭제하시겠습니까?" ✓
  - "삭제" 확인 후 목록에서 항목 제거 ✓
  - 목록 비었을 때 "할일이 없습니다." 표시 ✓

---

### SCN-12: 할일 목록 필터링 조회

- **결과**: ✅ PASS
- **검증 항목**
  - **카테고리 필터**: "업무" 선택 → URL `?categoryId={id}` 파라미터 반영 ✓, 업무 할일만 표시 ✓
  - **기간 만료 필터**: 체크 → URL `&overdue=true` 추가 ✓
  - **기간 만료 배지**: "업무 주간보고 작성" 항목에 "기간 만료" 배지 표시 ✓ (종료일 `2026-05-14` < 오늘)
  - BR-08 필터 조건 조합 동작 확인

### SCN-13: 카테고리 추가

  - 목록에 "사이드프로젝트" 추가됨 ✓

### SCN-14: 카테고리 중복명 등록 실패

- **결과**: ✅ PASS
- **검증 항목**
  - "사이드프로젝트" 중복 입력 후 "추가" 클릭
  - 서버 409 Conflict 응답 (콘솔 에러 확인)
  - 목록에 중복 항목 미생성 확인 ✓
- **비고**: 오류 토스트 빠르게 사라짐. 목록 불변 + 콘솔 에러로 동작 확인.

---

### SCN-15: 카테고리 삭제 및 할일 자동 재배정

- **결과**: ✅ PASS
- **검증 항목**
  - "사이드프로젝트" 카테고리 소속 할일("사이드 앱 기획서") 사전 등록
  - 삭제 버튼 클릭 → 확인 다이얼로그 표시
  - 다이얼로그 메시지: "소속 할일은 기본 카테고리로 이동됩니다." ✓
  - 삭제 확정 후 "사이드프로젝트" 카테고리 목록에서 제거 ✓
  - "사이드 앱 기획서" 할일이 기본 카테고리("개인")로 재배정됨 ✓ (BR-07)

---

### SCN-16: 기본 카테고리 삭제 시도 실패

- **결과**: ✅ PASS
- **검증 항목**

- **결과**: ✅ PASS
- **검증 항목**

  - `/login` 페이지로 리다이렉트 ✓

### SCN-18: 회원 탈퇴

- **결과**: ✅ PASS
- **검증 항목**
  - `/profile` 하단 "회원 탈퇴" 버튼 클릭

  - **2단계 다이얼로그**: 삭제 항목 목록 표시 (계정·할일·카테고리) ✓, "이 작업은 절대 되돌릴 수 없습니다." 경고 ✓
  - "탈퇴하기" 확정 후 모든 데이터 삭제
  - `/login` 리다이렉트 ✓ (NFR-04 트랜잭션 처리)
